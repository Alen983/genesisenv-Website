# Corporate Systems

Enterprise features and team workflows for genesis-env.

## Team Workflows

### Shared Templates

Maintain a central `.env.template` file in your repository:

```bash
# .env.template (committed to git)
DATABASE_URL=required
API_KEY=required
REDIS_URL=optional
```

Each developer creates their own `.env` from the template.

### Validation in CI/CD

Ensure all environments match the template:

```yaml
# .github/workflows/validate-env.yml
name: Validate Environment
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Validate .env.template
        run: genesis-env validate .env.template --strict
```

## Getting Started

1. Create a `.env.template` file in your repository
2. Add validation to your CI/CD pipeline
3. Document required keys for your team
4. Use the playground to test templates before committing
