# Templates & `.env`

How genesis-env uses your committed template and your local secrets file. There is **no** `validate` subcommand in the current CLI; this page describes **generate** (including the default `npx genesis-env` flow) and **init**.

## Concepts

| Concept | File | Role |
|--------|------|------|
| Template (committed) | `.env.template` or `.env.example` | Lists keys and optional defaults; safe to commit to git. |
| Local secrets | `.env` | Filled by **generate**; should stay local and gitignored; **init** helps add `.env` to `.gitignore`. |

If both **`.env.template`** and **`.env.example`** exist in the project root, **`.env.template` is used**.

The genesis-env CLI does **not** require you to set any environment variables on your machine to run it. The variables you care about are the ones your **application** defines in its template.

## How generate uses the template

The CLI reads the template from the **project root**, prompts once per variable (using optional defaults from `KEY=value` lines), then writes **`.env`**. It refuses to overwrite an existing **`.env`** without **`--force` / `-f`** (documented under **Options / flags** on the [Commands](/docs/commands) page). Command forms (`npx genesis-env`, `generate`, etc.) are on that page too.

**Line rules:** comments and blank lines are skipped. Each other line should define a variable, optionally with a default:

```bash
# Comments are ignored
DATABASE_URL=
API_KEY=changeme
PORT=3000
```

Prompt text comes from each variable‚Äôs name and optional default (the part after `=` on each line).

## Template requirements

- The template must live in the **root directory** (where you run the CLI).
- The file must be **UTF-8** encoded (UTF-16 is not supported).

## `init` (scaffold)

Use **`npx genesis-env init`** when you need missing `.env` / `.env.template` files or a **`.gitignore`** entry for `.env`. Full syntax is under **init** on the [Commands](/docs/commands) page. Typical messages include `.env created`, `.env.template created`, or `.env already ignored`, depending on what already exists.

## Quick example: generate

```bash
$ npx genesis-env

Found .env.template

? DATABASE_URL:
? API_KEY (default: changeme):
? PORT (default: 3000):

Created .env
```

To overwrite an existing `.env`, use **`--force` / `-f`** as on the [Commands](/docs/commands) page.

## Why this flow

- **Saves time** ‚Äî no manual copy-and-edit of example env files.
- **Fewer missing keys** ‚Äî you walk through every key in the template before `.env` is written.
- **Easier onboarding** ‚Äî new contributors follow the same one-command flow.
- **Runs locally** ‚Äî no network calls; nothing you type is sent anywhere.
